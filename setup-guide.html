Getting started

How to Set Up Arkoris

Get your Discord server and ARK cluster connected in a few simple steps. Most features are optional and can be enabled later — start with the basics and grow from there.

Start Setup Wizard Open Dashboard Run Setup Check

In Discord, run /setup → 🚀 Setup Wizard to do all of this guided, in order.

The 9 steps

From invite to live in minutes

Each step explains what you're configuring and why. Required steps are marked; everything else is optional.

1

Invite Arkoris & check permissions · required

Add the bot, then make sure its role sits above the roles it needs to manage, and that it can see and post in your channels.

  • ✅ Bot added to Discord
  • ✅ Bot role positioned correctly (drag it high in Server Settings → Roles)
  • ✅ Required permissions available · slash commands visible
2

Set owner, admin & staff roles · required

Owner roles control sensitive actions (wipes, resets, payments). Admin roles reach /setup and /ark. Staff roles handle tickets, moderation, polls, events, giveaways, grace timers and Cluster Alpha.

Configure in /setup → 🎫 Tickets & Staff → Staff Roles & Permissions.

3

Connect Nitrado · optional (ARK features)

Paste your Nitrado token in /setup → 🦖 ARK Integration → Nitrado Connection, validate it, then link each ARK map. Game-chat relay and population tracking are configured automatically — no manual setup.

Your token is encrypted at rest and never shown again after you save it.

4

Generate recommended channels · optional

Preview, then one-click create: Discord audit logs, ARK forum logs, ticket categories, staff earnings forum, population board and announcements. Arkoris detects what already exists, so it never makes duplicates.

/setup → 📋 Logs & Monitoring → Generate Logs (preview first).

5

Configure tickets · optional

Set the panel channel, staff roles, ticket categories, questions and transcripts in /setup → 🎫 Tickets & Staff → Ticket System, then publish the panel. Every ticket gets a control row:

[ Add Member ] [ Rename Ticket ] [ Close Ticket ]
[ Log Work ] [ Create Payment Link ] [ Export Transcript ]

Staff rename; staff or the ticket owner close. Add Member, Log Work and Payment are staff-only.

6

Publish the Player Hub · optional

A one-click panel your members use without learning commands: Population · My Rank · My Credits · Leaderboard · Submit Suggestion · Open Ticket.

7

Optional community features

Enable any of: welcome messages, auto-roles, levels & XP, credits, hype rewards, suggestions, polls, events, giveaways, sticky messages, role menus and Cluster Alpha — all under /setup → 🏠 Discord Server.

8

Optional payments & branding · premium

Subscription, premium branding, PayPal/Stripe and the public directory live under /setup → 💳 Payments & Branding. Enter payment secrets on the web dashboard — never in a public Discord channel.

Webhook endpoint (auto-confirms payments): the Payments panel shows your bot's Webhook URL for each provider. Copy it and add it as an endpoint:
• Stripe → Developers → Webhooks → Add endpoint → paste …/webhooks/stripe, select event checkout.session.completed, then paste the whsec_… Signing secret back into /setup.
• PayPal → Developer Dashboard → Webhooks → paste …/webhooks/paypal, subscribe to PAYMENT.CAPTURE.COMPLETED, then paste the Webhook ID back into /setup.

9

Run the final setup check · recommended

/setup → ✅ Review Configuration validates permissions, roles, channels, ticket panel, Nitrado token, linked maps, chat relay, population tracking, ARK logs, webhooks, background jobs and published panels.

Results show ✅ working · ⚠ recommended · ❌ must fix · ⚪ optional.

Troubleshooting

Common issues & quick fixes

Bot commands are missing

The bot was invited without the applications.commands scope, or commands haven't synced yet. Re-invite with the link on this site and wait a minute.

Bot cannot create channels

Its role is too low or lacks Manage Channels. Drag the Arkoris role higher in Server Settings → Roles.

Tickets do not work

Publish the ticket panel and set staff roles in Tickets & Staff. Use Permission Repair if access looks wrong.

Nitrado connection failed

Re-paste the token in ARK Integration → Nitrado Connection and click Validate. Tokens expire — generate a fresh one if needed.

Chat relay is offline

Check the relay channel in ARK Integration → Game-Chat Relay and that the linked map is online.

Population board not updating

Re-publish the board from ARK Integration → Population & Live Boards; confirm the map is linked and online.

Payment provider incomplete

Finish PayPal/Stripe config on the dashboard (client id, secret, webhook). The Review check flags exactly what's missing.

Webhook is failing

Confirm the webhook URL is set in your PayPal/Stripe dashboard and matches the one shown in Payment Providers.

FAQ

Quick answers

What's required to start?

Just invite the bot and set your roles. Everything else is optional.

Is Nitrado required?

Only for ARK features (player tracking, logs, wipes, ARK Guard). The community tools work without it.

Are PayPal & Stripe optional?

Yes — only needed if you take payments. Enter secrets on the dashboard.

Can I configure features later?

Yes. Enable modules any time from /setup; nothing has to be done up front.

Who can use /setup?

Guild owner and configured admin roles only.

Who can use /ark?

Admins; sensitive actions (wipes, password, cluster id, resets) are owner-only.

Can regular players view credits?

Yes — their own balance and history via /credit balance and /credit history. Only staff can give or remove.

Who can rename a ticket?

Staff only. Ticket owners can close their ticket but not rename it.

How do I reset a season?

/ark → ♻ Season & Data — owner-only, with typed confirmation and an audit log.

How do I wipe ARK maps safely?

/ark → 🎛 Server Operations → Wipe Management. Use Dry run first — it shows exactly what would change (one shared cluster id across selected maps) with no changes made.
